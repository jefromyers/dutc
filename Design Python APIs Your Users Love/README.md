# Problem: Programmatic Documentation


### Lab 1

1. Some code needs to be run daily that requires several outside services and
   you would like to have Slack confirmation on the status of completion or
   failure (and where it failed).

2. Some code was written in a substandard fashion because a library was
   out-of-date and should be revisited once the library is either updated or
   when it reaches a particular version.

3. Some code was written as a quick workaround to address an immediately
   problem and should be revisited within a particular time-window. If not
   revisited within a particular time-window, it should generate warnings or
   even refuse to run.

4. Some code or some function was written by copying-and-pasting from the
   documentation or “quick start” page and should be revisited if the original
   version changes in some fashion.
